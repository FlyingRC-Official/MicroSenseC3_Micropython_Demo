# SPA06-003 AI Coding Notes

## What this device is
SPA06-003 is a digital barometric pressure sensor with integrated temperature sensing. It supports I2C and SPI, provides 24-bit raw outputs, and requires host-side compensation using factory calibration coefficients.

## Core capabilities
- Pressure + temperature sensing
- Pressure range: `300 .. 1100 hPa`
- Temperature range: `-40 .. +85 C`
- Supply:
  - `VDD`: 1.7 V to 3.6 V
  - `VDDIO`: 1.08 V to 3.6 V
- 24-bit raw pressure result
- 24-bit raw temperature result
- FIFO depth: 32 measurements
- I2C or SPI interface
- Default I2C address: `0x77`
- Alternate I2C address: `0x76` when `SDO` is pulled low
- ID register `0x0D`, reset value `0x11`
- Relative accuracy: about `±0.03 hPa`
- Absolute accuracy: about `±0.3 hPa`

## Important high-level rule
This is **not** a simple raw-value sensor.

For correct pressure and temperature output you generally need to:
1. Wait for `SENSOR_RDY` and `COEF_RDY`
2. Read calibration coefficients from `0x10..0x24`
3. Read raw 24-bit pressure and/or temperature
4. Apply datasheet compensation formulas

## Hardware integration notes
### Relevant pins
- `1` `GND`
- `2` `CSB`
- `3` `SDA` - serial data in/out in I2C and 3-wire SPI, serial data in in 4-wire SPI
- `4` `SCK` - serial clock
- `5` `SDO` - SPI data out or I2C address LSB / interrupt-related pin depending on mode
- `6` `VDDIO`
- `7` `GND`
- `8` `VDD`

### I2C address selection
- `0x77` default
- `0x76` if `SDO` pulled down to GND

### Interrupt pin note
In I2C mode, `SDO` can also be used as interrupt output, so board design must account for address selection vs interrupt usage.

## Modes of operation
### Standby mode
- Default after power-on or reset
- No measurements performed
- Registers and coefficients accessible

### Command mode
- One pressure or temperature measurement
- Sensor returns to standby after conversion

### Background mode
- Continuous measurements
- Can run pressure, temperature, or both
- FIFO can store the latest 32 results

## Register map
### Raw data
- `0x00` `PRS_B2`
- `0x01` `PRS_B1`
- `0x02` `PRS_B0`
- `0x03` `TMP_B2`
- `0x04` `TMP_B1`
- `0x05` `TMP_B0`

### Configuration
- `0x06` `PRS_CFG`
- `0x07` `TMP_CFG`
- `0x08` `MEAS_CFG`
- `0x09` `CFG_REG`
- `0x0A` `INT_STS`
- `0x0B` `FIFO_STS`
- `0x0C` `RESET`
- `0x0D` `ID`
- `0x10..0x24` `COEF`

## Register details for coding
### `0x06 PRS_CFG`
#### `PM_RATE[3:0]`
Pressure rate in background mode:
- `0000` 1 / s
- `0001` 2 / s
- `0010` 4 / s
- `0011` 8 / s
- `0100` 16 / s
- `0101` 32 / s
- `0110` 64 / s
- `0111` 128 / s
- `1000` 25/16 / s
- `1001` 25/8 / s
- `1010` 25/4 / s
- `1011` 25/2 / s
- `1100` 25 / s
- `1101` 50 / s
- `1110` 100 / s
- `1111` 200 / s

#### `PM_PRC[3:0]`
Pressure oversampling:
- `0000` single
- `0001` 2x
- `0010` 4x
- `0011` 8x
- `0100` 16x
- `0101` 32x
- `0110` 64x
- `0111` 128x

### `0x07 TMP_CFG`
Same concept for temperature:
- rate in upper nibble
- precision/oversampling in lower nibble

### `0x08 MEAS_CFG`
Status + mode control:
- bit 7 `COEF_RDY`
- bit 6 `SENSOR_RDY`
- bit 5 `TMP_RDY`
- bit 4 `PRS_RDY`
- bit 3 `TMP_EXT`
- bits 2:0 `MEAS_CTRL`

`MEAS_CTRL` values:
- `000` standby
- `001` one-shot pressure
- `010` one-shot temperature
- `101` continuous pressure
- `110` continuous temperature
- `111` continuous pressure + temperature

### `0x09 CFG_REG`
- bit 7 `INT_HL`
- bit 6 `INT_FIFO`
- bit 5 `INT_TMP`
- bit 4 `INT_PRS`
- bit 3 `T_SHIFT`
- bit 2 `P_SHIFT`
- bit 1 `FIFO_EN`
- bit 0 `SPI_MODE`

Important note:
- `T_SHIFT` and `P_SHIFT` must be set to `1` when oversampling is greater than `8x`

### `0x0C RESET`
- bit 7 `FIFO_FLUSH` -> write `1` to flush FIFO
- bits 3:0 `SOFT_RST` -> write binary `1001` to trigger soft reset

### `0x0D ID`
- upper nibble = revision ID
- lower nibble = product ID
- reset value shown as `0x11`

## Raw data format
Pressure and temperature results are 24-bit signed 2's complement.

Combine helper:
```c
int32_t spa06_sign_extend24(uint32_t x) {
    if (x & 0x800000u) x |= 0xFF000000u;
    return (int32_t)x;
}

int32_t spa06_read24(uint8_t msb, uint8_t mid, uint8_t lsb) {
    uint32_t raw = ((uint32_t)msb << 16) | ((uint32_t)mid << 8) | lsb;
    return spa06_sign_extend24(raw);
}
```

## Calibration coefficients
Calibration coefficients are stored in `0x10..0x24` and are bit-packed.

Coefficient widths:
- `c0`, `c1`: 12-bit signed
- `c00`, `c10`: 20-bit signed
- `c20`, `c30`, `c01`, `c11`, `c21`: 16-bit signed
- `c31`, `c40`: 12-bit signed

Packing overview from datasheet:
- `0x10` -> `c0[11:4]`
- `0x11` -> `c0[3:0]` and part of `c1`
- `0x12` -> `c1[7:0]`
- `0x13` -> `c00[19:12]`
- `0x14` -> `c00[11:4]`
- `0x15` -> `c00[3:0]` and part of `c10`
- `0x16` -> `c10[15:8]`
- `0x17` -> `c10[7:0]`
- `0x18..0x21` -> `c01`, `c11`, `c20`, `c21`, `c30`
- `0x22..0x24` -> `c31`, `c40`

This is the part most likely to cause bugs, so it is worth isolating into a dedicated unpack function.

## Compensation scale factors
Use datasheet scale factor `kP` / `kT` based on oversampling:
- 1x -> `524288`
- 2x -> `1572864`
- 4x -> `3670016`
- 8x -> `7864320`
- 16x -> `253952`
- 32x -> `516096`
- 64x -> `1040384`
- 128x -> `2088960`

## Compensation formulas
### Temperature
```c
Traw_sc = Traw / kT;
Tcomp_C = c0 * 0.5f + c1 * Traw_sc;
```

### Pressure
```c
Traw_sc = Traw / kT;
Praw_sc = Praw / kP;

Pcomp_Pa = c00
         + c10 * Praw_sc
         + c20 * Praw_sc * Praw_sc
         + c30 * Praw_sc * Praw_sc * Praw_sc
         + c40 * Praw_sc * Praw_sc * Praw_sc * Praw_sc
         + Traw_sc * (c01
                    + c11 * Praw_sc
                    + c21 * Praw_sc * Praw_sc
                    + c31 * Praw_sc * Praw_sc * Praw_sc);
```

## Practical init flow
### Minimum safe bring-up
1. Power sensor
2. Poll `MEAS_CFG` until:
   - `SENSOR_RDY = 1`
   - `COEF_RDY = 1`
3. Read ID register `0x0D`
4. Read and unpack coefficients from `0x10..0x24`
5. Configure `PRS_CFG`
6. Configure `TMP_CFG`
7. Configure `CFG_REG`
8. Start command or background mode through `MEAS_CFG`

### Simple background example
A sensible starter setup:
- pressure oversampling `16x` or `32x`
- temperature oversampling `1x` or `2x`
- continuous pressure + temperature mode
- enable `P_SHIFT` when oversampling > 8x

## Example pseudo-code
```c
bool spa06_init(void) {
    delay_ms(5);

    // Wait for startup complete
    while (1) {
        uint8_t meas = i2c_read_reg(SPA06_ADDR, 0x08);
        bool coef_rdy   = (meas & 0x80) != 0;
        bool sensor_rdy = (meas & 0x40) != 0;
        if (coef_rdy && sensor_rdy) break;
    }

    uint8_t id = i2c_read_reg(SPA06_ADDR, 0x0D);
    (void)id; // expected reset value documented as 0x11

    // Example: pressure 16x OSR at 2/s -> 0x14
    i2c_write_reg(SPA06_ADDR, 0x06, 0x14);

    // Example: temperature single-shot rate field at 1/s, single precision -> 0x00
    i2c_write_reg(SPA06_ADDR, 0x07, 0x00);

    // Enable pressure shift because pressure OSR > 8x
    i2c_write_reg(SPA06_ADDR, 0x09, 0x04);

    // Start continuous pressure + temperature
    i2c_write_reg(SPA06_ADDR, 0x08, 0x07);

    return true;
}
```

## Reading raw results
```c
int32_t p_raw = spa06_read24(buf[0], buf[1], buf[2]);
int32_t t_raw = spa06_read24(buf[3], buf[4], buf[5]);
```

Then run compensation.

## FIFO behavior
- FIFO stores the latest 32 measurements
- FIFO may contain pressure and temperature results intermixed
- Least significant bit indicates measurement type:
  - `1` pressure
  - `0` temperature
- Data is always read out through the pressure data registers `0x00..0x02`
- If FIFO becomes empty, further reads return `0x800000`

## Useful formulas for application layer
### Altitude from pressure
```c
altitude_m = 44330.0f * (1.0f - powf(P / 1013.25f, 1.0f / 5.255f));
```

### Sea-level pressure from altitude
```c
p0 = p / powf(1.0f - altitude_m / 44330.0f, 5.255f);
```

## Important driver gotchas
- Do not use raw pressure directly without coefficient compensation.
- Wait for both `SENSOR_RDY` and `COEF_RDY` after startup.
- Set `P_SHIFT` / `T_SHIFT` when oversampling is greater than 8x.
- Temperature should be measured alongside pressure for correct pressure compensation.
- FIFO output multiplexes pressure and temperature; do not assume every FIFO word is pressure.

Implement these functions:

- `soft_reset()`
- `read_id()`
- `read_coefficients()`

- `configure_temperature(rate, osr)`
- `set_mode(mode)`
- `read_raw_pt()`
- `compensate_temperature()`
- `compensate_pressure()`
- `read_pressure_pa()`
- `read_temperature_c()`

## Best first driver target
For first bring-up, do this in order:
1. one-shot pressure read
2. one-shot temperature read
3. coefficient unpacking
4. compensated temperature
5. compensated pressure
6. only then background mode and FIFO

That sequence avoids most debugging confusion.
