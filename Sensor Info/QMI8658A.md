# QMI8658A AI Coding Notes

## What this device is
QMI8658A is a 6-axis IMU with a 3-axis accelerometer and 3-axis gyroscope. It also includes a temperature sensor, FIFO, interrupt support, and embedded motion features such as tap, pedometer, motion detection, and wake-on-motion.

## Core capabilities
- 3-axis accelerometer + 3-axis gyroscope
- Temperature output
- Interfaces: I2C, I3C, 3-wire SPI, 4-wire SPI
- FIFO size: 1536 bytes
- Dual interrupt pins: `INT1`, `INT2`
- Accelerometer ranges: ±2 g, ±4 g, ±8 g, ±16 g
- Gyroscope ranges: ±16 dps to ±2048 dps
- Operating temperature: -40 C to +85 C
- `WHO_AM_I` register at `0x00`, expected value `0x05`
- Soft reset register at `0x60`, write `0xB0`

## Hardware integration notes
### I2C addresses
Selected by `SA0` / `SDO/SA0` pin:
- `0x6A` when SA0 is high or left floating in I2C mode
- `0x6B` when SA0 is low

### Pins that matter most for common firmware
- `INT1`, `INT2` - interrupts
- `VDDIO` - digital I/O supply
- `VDD` - analog/core supply
- `SCL`, `SDA` - I2C
- `CS` - bus mode select
- `SDO/SA0` - I2C address select or SPI data out

### Power-up cautions
- System turn-on time is about `15 ms` from reset/no-power to power-on default state
- If `VDD` and `VDDIO` are separate, `VDDIO` must be powered **no later than** `VDD`
- Datasheet warns the ramp from about `1.1 V` to `1.71 V` should have slew rate above `40 V/s`

Practical rule:
1. Power rails up correctly
2. Wait at least `15 ms`
3. Read `WHO_AM_I`
4. Configure accel/gyro
5. Enable sensors in `CTRL7`
6. Poll status or use DRDY/FIFO interrupts

## Important registers
### Identification
- `0x00` `WHO_AM_I` -> expected `0x05`
- `0x01` `REVISION_ID`

### Main configuration
- `0x02` `CTRL1`
- `0x03` `CTRL2` accel settings
- `0x04` `CTRL3` gyro settings
- `0x06` `CTRL5` LPF settings
- `0x08` `CTRL7` sensor enable / DRDY / sync sample
- `0x09` `CTRL8` motion-engine enables
- `0x0A` `CTRL9` command register

### FIFO
- `0x13` `FIFO_WTM_TH`
- `0x14` `FIFO_CTRL`
- `0x15` `FIFO_SMPL_CNT`
- `0x16` `FIFO_STATUS`
- `0x17` `FIFO_DATA`

### Status
- `0x2D` `STATUSINT`
- `0x2E` `STATUS0`
- `0x2F` `STATUS1`
- `0x30..0x32` `TIMESTAMP`

### Data output
- `0x33..0x34` `TEMP_L/H`
- `0x35..0x3A` `AX/AY/AZ`
- `0x3B..0x40` `GX/GY/GZ`

### Reset
- `0x60` `RESET` -> write `0xB0`

## Register details for coding
### `0x02 CTRL1`
- bit 7 `SIM`
  - `0` 4-wire SPI
  - `1` 3-wire SPI
- bit 6 `ADDR_AI`
  - `0` no auto-increment
  - `1` address auto-increment
- bit 5 `BE`
  - `0` little-endian read data
  - `1` big-endian read data
- bit 4 `INT2_EN`
- bit 3 `INT1_EN`
- bit 2 `FIFO_INT_SEL`
  - `0` FIFO interrupt on INT2
  - `1` FIFO interrupt on INT1
- bit 0 `SensorDisable`
  - `0` enable internal high-speed oscillator
  - `1` disable internal high-speed oscillator

Practical recommendation:
- Set `ADDR_AI = 1` for burst reads
- Leave `BE` alone unless your existing driver architecture requires a specific byte ordering mode

### `0x03 CTRL2` accelerometer settings
- bit 7 `aST` accel self-test enable
- bits 6:4 `aFS`
  - `000` ±2 g
  - `001` ±4 g
  - `010` ±8 g
  - `011` ±16 g
- bits 3:0 `aODR`
  - accel-only low-power options at lower codes near `1100..1111`
  - common normal-mode values:
    - `0011` -> 1000 Hz
    - `0100` -> 500 Hz
    - `0101` -> 250 Hz
    - `0110` -> 125 Hz
    - `0111` -> 62.5 Hz
    - `1000` -> 31.25 Hz

### `0x04 CTRL3` gyroscope settings
- bit 7 `gST` gyro self-test enable
- bits 6:4 `gFS`
- bits 3:0 `gODR`
  - `0000` -> 7174.4 Hz
  - `0001` -> 3587.2 Hz
  - `0010` -> 1793.6 Hz
  - `0011` -> 896.8 Hz
  - `0100` -> 448.4 Hz
  - `0101` -> 224.2 Hz
  - `0110` -> 112.1 Hz
  - `0111` -> 56.05 Hz
  - `1000` -> 28.025 Hz

### `0x06 CTRL5` LPF settings
- bit 4 `gLPF_EN`
- bits 6:5 `gLPF_MODE`
- bit 0 `aLPF_EN`
- bits 2:1 `aLPF_MODE`

LPF bandwidth fractions listed by datasheet:
- `00` -> 2.66% of ODR
- `01` -> 3.63% of ODR
- `10` -> 5.39% of ODR
- `11` -> 13.37% of ODR

### `0x08 CTRL7`
- bit 7 `SyncSample`
- bit 5 `DRDY_DIS`
- bit 4 `gSN` gyro snooze/full mode
- bit 1 `gEN`
- bit 0 `aEN`

Common values:
- `0x01` -> accel only
- `0x02` -> gyro only
- `0x03` -> accel + gyro
- `0x81`, `0x82`, `0x83` -> same, but with SyncSample enabled

### `0x09 CTRL8`
Enables embedded engines:
- bit 4 pedometer
- bit 3 significant motion
- bit 2 no motion
- bit 1 any motion
- bit 0 tap

### `0x2E STATUS0`
- bit 1 `gDA` -> new gyro data available
- bit 0 `aDA` -> new accel data available

### `0x2F STATUS1`
- bit 7 significant motion
- bit 6 no motion
- bit 5 any motion
- bit 4 pedometer
- bit 2 wake on motion
- bit 1 tap

## Data format
### Temperature
Registers `0x33` and `0x34`
```c
float temperature_c = (float)temp_h + ((float)temp_l / 256.0f);
```

### Acceleration and angular rate
All axis outputs are signed 16-bit two's complement.

```c
static inline int16_t qmi8658_combine(uint8_t lo, uint8_t hi) {
    return (int16_t)((uint16_t)lo | ((uint16_t)hi << 8));
}
```

Convert to physical units:
```c
float accel_g = raw_accel / accel_lsb_per_g;
float gyro_dps = raw_gyro / gyro_lsb_per_dps;
```

## Basic init sequence
A practical baseline driver should:
1. Wait for power-up
2. Read `WHO_AM_I` and verify `0x05`
3. Optionally set `CTRL1.ADDR_AI = 1` for burst reads
4. Configure `CTRL2` accel range + ODR
5. Configure `CTRL3` gyro range + ODR
6. Optionally configure `CTRL5` LPF
7. Write `CTRL7` to enable accel, gyro, or both
8. Poll `STATUS0` or use DRDY/FIFO interrupts
9. Read `TEMP`, `ACCEL`, `GYRO`

Example baseline choices:
- accel: ±4 g at 250 Hz
- gyro: ±512 dps at 224.2 Hz
- LPF enabled for both
- accel + gyro enabled in `CTRL7`

## Example probe and init pseudo-code
```c
bool qmi8658_init(uint8_t addr) {
    delay_ms(20);

    uint8_t who = i2c_read_reg(addr, 0x00);
    if (who != 0x05) return false;

    // CTRL1: auto-increment on, keep other bits conservative
    i2c_write_reg(addr, 0x02, (1u << 6));

    // CTRL2: accel ±4g, 250 Hz
    // aFS=001, aODR=0101
    i2c_write_reg(addr, 0x03, (1u << 4) | 0x05);

    // CTRL3: gyro ±512 dps, 224.2 Hz
    // gFS=101, gODR=0101
    i2c_write_reg(addr, 0x04, (5u << 4) | 0x05);

    // CTRL5: enable LPF for accel and gyro, mode 00
    i2c_write_reg(addr, 0x06, (1u << 4) | (1u << 0));

    // CTRL7: enable accel + gyro
    i2c_write_reg(addr, 0x08, 0x03);

    return true;
}
```

## Burst read example
```c
bool qmi8658_read_6dof(uint8_t addr,
                       int16_t *ax, int16_t *ay, int16_t *az,
                       int16_t *gx, int16_t *gy, int16_t *gz,
                       float *temp_c) {
    uint8_t st = i2c_read_reg(addr, 0x2E);
    if ((st & 0x03) == 0) return false;

    uint8_t buf[14];
    i2c_read_regs(addr, 0x33, buf, sizeof(buf));

    uint8_t temp_l = buf[0];
    uint8_t temp_h = buf[1];
    *temp_c = (float)temp_h + ((float)temp_l / 256.0f);

    *ax = qmi8658_combine(buf[2],  buf[3]);
    *ay = qmi8658_combine(buf[4],  buf[5]);
    *az = qmi8658_combine(buf[6],  buf[7]);
    *gx = qmi8658_combine(buf[8],  buf[9]);
    *gy = qmi8658_combine(buf[10], buf[11]);
    *gz = qmi8658_combine(buf[12], buf[13]);
    return true;
}
```

## FIFO basics
### FIFO registers
- `FIFO_WTM_TH` `0x13`
- `FIFO_CTRL` `0x14`
- `FIFO_SMPL_CNT` `0x15`
- `FIFO_STATUS` `0x16`
- `FIFO_DATA` `0x17`

### `FIFO_CTRL`
- bit 7 `FIFO_RD_MODE`
  - `0` write mode, sensors keep filling FIFO
  - `1` read mode, host reads from `FIFO_DATA`
- bits 3:2 `FIFO_SIZE`
  - `00` 16 samples
  - `01` 32 samples
  - `10` 64 samples
  - `11` 128 samples
- bits 1:0 `FIFO_MODE`
  - `00` bypass
  - `01` FIFO
  - `10` stream

### Important FIFO behavior
- To read FIFO, device expects a `CTRL9` command: `CTRL_CMD_REQ_FIFO (0x05)`
- While FIFO read mode is enabled, new samples are not filled into FIFO
- After FIFO readout, clear `FIFO_RD_MODE` back to `0`

## Reset behavior
Soft reset:
```c
i2c_write_reg(addr, 0x60, 0xB0);
delay_ms(20);
```

## CTRL9 commands worth knowing
- `0x00` `CTRL_CMD_ACK`
- `0x04` `CTRL_CMD_RST_FIFO`
- `0x05` `CTRL_CMD_REQ_FIFO`
- `0x08` configure wake-on-motion
- `0x0C` configure tap
- `0x0D` configure pedometer
- `0x0E` configure motion detection
- `0x12` AHB clock gating related
- `0xA2` on-demand calibration (COD)
- `0xAA` apply saved gyro gains

For first driver bring-up, you can ignore most CTRL9 features except FIFO reset / request.

## Self-test notes
### Accelerometer self-test
- Disable sensors: `CTRL7 = 0x00`
- Set `CTRL2.aST = 1`
- Wait for interrupt or `STATUSINT`
- Read `0x51..0x56`
- Absolute value of each axis should exceed about `200 mg`

### Gyroscope self-test
- Similar concept using `CTRL3.gST`
- The self-test routine automatically controls scale/ODR internally

## Calibration-On-Demand (COD)
This is an advanced feature for gyro X/Y gain calibration.

Basic flow:
1. Disable accel and gyro
2. Issue `CTRL9 = 0xA2`
3. Wait about `1.5 s`
4. Read `COD_STATUS` at `0x46`

For a first driver, this can be skipped.

## Important driver gotchas
- Respect power sequencing and ramp requirements.
- The device is feature-rich; start with plain register data before enabling motion engines.
- FIFO read mode is not just a simple burst read; it involves `CTRL9` handshake.
- `ADDR_AI` is useful for burst reads; if left at `0`, multi-byte transactions may not work the way your generic driver expects.
- DRDY and FIFO interrupts share logic with `INT1`/`INT2` configuration, so keep interrupt routing simple during bring-up.

## Recommended software API
Implement these first:
- `probe()`
- `reset()`
- `set_accel_config(fs, odr)`
- `set_gyro_config(fs, odr)`
- `set_lpf(...)`
- `enable_sensors(accel_on, gyro_on)`
- `read_status0()`
- `read_temperature()`
- `read_accel_raw()`
- `read_gyro_raw()`
- `read_6dof_raw()`
- `read_6dof_scaled()`
- `fifo_configure()`
- `fifo_read()`

## Best first driver target
Start with:
- I2C mode
- `WHO_AM_I` probe
- auto-increment enabled
- burst read of `0x33..0x40`
- accel + gyro only
- no motion engines
- no FIFO
- optional LPF

Once that is stable, add FIFO, then interrupts, then embedded motion features.
