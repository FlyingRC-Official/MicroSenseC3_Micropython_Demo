# QMC6309 AI Coding Notes

## What this device is
QMC6309 is a 3-axis magnetometer for compass and heading applications. It uses an I2C interface, outputs temperature-compensated magnetic data, and includes built-in self-test.

## Core capabilities
- 3-axis magnetic sensing
- I2C interface
- Supply: 2.5 V to 3.6 V
- Operating temperature: -40 C to +85 C
- Field range: ±32 G, ±16 G, ±8 G
- Output width: 16-bit signed 2's complement per axis
- Typical sensitivity:
  - ±32 G -> 1000 LSB/G
  - ±16 G -> 2000 LSB/G
  - ±8 G -> 4000 LSB/G
- Chip ID: `0x90`
- Built-in self-test in continuous mode

## Hardware integration notes
### Pins
- `A1` `VSS` - ground
- `A2` `SCL` - I2C clock
- `B1` `VDD` - supply
- `B2` `SDA` - I2C data

### External parts
- External I2C pull-ups are required
- Reservoir capacitor `C0` is nominally `2.2 uF`
- Datasheet recommends ceramic capacitor with low ESR, preferably below `200 mOhm`

### Layout cautions
- Keep ferrous materials away from the sensor
- Avoid copper routing directly under or very near the sensor on PCB layers
- This is important for heading accuracy

## Power-up behavior
- After power-on reset, the chip enters **Suspend Mode**
- Typical POR completion time to be ready for I2C commands: about `3 ms`
- Recommended power-off interval below the shutdown threshold before next POR: `>100 us`

Practical rule:
1. Apply power
2. Wait at least a few milliseconds
3. Read chip ID at `0x00`
4. Configure control registers
5. Start measurements

## I2C basics
- Only one documented 7-bit I2C address: `0x7C`
- I2C bus speed: `100 kHz` and `400 kHz`
- External pull-ups required

## Register map
### Identification
- `0x00` `CHIP_ID` -> expected value `0x90`

### Magnetic data
- `0x01` `XOUT_L`
- `0x02` `XOUT_H`
- `0x03` `YOUT_L`
- `0x04` `YOUT_H`
- `0x05` `ZOUT_L`
- `0x06` `ZOUT_H`

### Status
- `0x09` `STATUS`
  - bit 0 `DRDY` -> new XYZ data ready
  - bit 1 `OVFL` -> overflow occurred
  - bit 2 `ST_RDY` -> self-test done
  - bit 3 `NVM_RDY`
  - bit 4 `NVM_LOAD_DONE`

### Control
- `0x0A` `CTRL1`
  - bits 1:0 `MODE`
  - bits 3:2 `OSR1`
  - bits 7:5 `OSR2`
- `0x0B` `CTRL2`
  - bit 7 `SOFT_RST`
  - bits 6:4 `ODR`
  - bits 3:2 `RNG`
  - bits 1:0 `SET_RESET_MODE`
- `0x0E` `CTRL3`
  - bit 7 `SELFTEST`
- `0x13` `XST`
- `0x14` `YST`
- `0x15` `ZST`

## Register details for coding
### `0x0A CTRL1`
#### `MODE[1:0]`
- `00` -> Suspend
- `01` -> Normal
- `10` -> Single
- `11` -> Continuous

#### `OSR1[1:0]`
- `00` -> 8
- `01` -> 4
- `10` -> 2
- `11` -> 1

#### `OSR2[2:0]`
Low-pass filter depth:
- `000` -> 1
- `001` -> 2
- `010` -> 4
- `011` -> 8
- `100..111` -> 16

### `0x0B CTRL2`
#### `ODR[2:0]`
- `000` -> 1 Hz
- `001` -> 10 Hz
- `010` -> 50 Hz
- `011` -> 100 Hz
- `100..111` -> 200 Hz

#### `RNG[1:0]`
- `00` -> ±32 G
- `01` -> ±16 G
- `10` -> ±8 G
- `11` -> ±32 G

#### `SET_RESET_MODE[1:0]`
- `00` -> set and reset on
- `01` -> set only on
- `11` -> set and reset off

#### `SOFT_RST`
- Write `0x80` to request reset
- Then write `0x00` again, because the reset bit does **not** auto-clear

## Data format
Each axis is a signed 16-bit 2's complement value.

Recommended combine logic:
```c
int16_t qmc6309_combine_u16(uint8_t lo, uint8_t hi) {
    return (int16_t)((uint16_t)lo | ((uint16_t)hi << 8));
}
```

Example:
```c
int16_t x = qmc6309_combine_u16(buf[0], buf[1]);
int16_t y = qmc6309_combine_u16(buf[2], buf[3]);
int16_t z = qmc6309_combine_u16(buf[4], buf[5]);
```

## Converting raw data to Gauss
Use the selected full-scale sensitivity:
```c
float qmc6309_lsb_per_gauss(uint8_t rng_bits) {
    switch (rng_bits & 0x03) {
        case 0x00: return 1000.0f; // ±32 G
        case 0x01: return 2000.0f; // ±16 G
        case 0x02: return 4000.0f; // ±8 G
        default:   return 1000.0f;
    }
}

float gauss = raw / qmc6309_lsb_per_gauss(rng_bits);
```

## Modes of operation
### Suspend mode
- Default after POR and soft reset
- Lowest power
- No measurements running
- Registers remain accessible

### Normal mode
- Repeated measurements at configured ODR
- Best default mode for ordinary polling

### Single mode
- Performs one measurement, then returns to suspend
- Good for low-duty-cycle sampling

### Continuous mode
- Runs continuously without sleep gaps
- Maximum throughput
- Required for self-test

## Recommended init sequences
### Simple normal-mode init
Datasheet example:
- Write `0x40` to `0x0B`
- Write `0x61` to `0x0A`

Interpretation:
- Set/reset on
- Range ±32 G
- ODR 200 Hz
- Normal mode
- OSR1 = 8
- OSR2 = 8

### Simple continuous-mode init
Datasheet example:
- Write `0x00` to `0x0B`
- Write `0x63` to `0x0A`

## Practical init flow for firmware
```c
bool qmc6309_init(void) {
    delay_ms(5);

    uint8_t id = i2c_read_reg(QMC6309_ADDR, 0x00);
    if (id != 0x90) return false;

    // Example: set/reset on, ±32 G, 200 Hz
    i2c_write_reg(QMC6309_ADDR, 0x0B, 0x40);

    // Normal mode, OSR1=8, OSR2=8
    i2c_write_reg(QMC6309_ADDR, 0x0A, 0x61);

    return true;
}
```

## Reading measurements
```c
bool qmc6309_read_xyz(int16_t *x, int16_t *y, int16_t *z) {
    uint8_t st = i2c_read_reg(QMC6309_ADDR, 0x09);
    if ((st & 0x01) == 0) return false; // DRDY

    uint8_t buf[6];
    i2c_read_regs(QMC6309_ADDR, 0x01, buf, 6);

    *x = (int16_t)((uint16_t)buf[0] | ((uint16_t)buf[1] << 8));
    *y = (int16_t)((uint16_t)buf[2] | ((uint16_t)buf[3] << 8));
    *z = (int16_t)((uint16_t)buf[4] | ((uint16_t)buf[5] << 8));
    return true;
}
```

## Self-test flow
Self-test only works in **Continuous Mode**.

Datasheet procedure:
1. Write `0x00` to `0x0A` -> Suspend
2. Write `0x03` to `0x0A` -> Continuous
3. Wait about `20 ms`
4. Write `0x80` to `0x0E` -> enable self-test
5. Wait about `150 ms`
6. Poll `STATUS[2]` (`ST_RDY`)
7. Read `0x13` to `0x15`

Expected self-test range per axis:
- `-50` to `-1` LSB

## Important driver gotchas
- The chip powers up in suspend mode, so reading data immediately after power-up will not produce live measurements.
- `SOFT_RST` does not auto-clear; always write `0x00` after issuing reset.
- `OVFL` indicates at least one axis exceeded safe code range.
- If you switch between active modes, it is safest to go through **Suspend Mode**.
- Raw axis values are already temperature compensated according to the datasheet.

## Best first driver target
For a first stable driver, use:
- I2C burst read of `0x01..0x06`
- Normal mode
- ±32 G range
- 50 Hz or 100 Hz ODR
- OSR1 = 8
- OSR2 = 8

That gives a simple and reliable baseline before adding calibration, tilt compensation, or heading estimation.
